# TreeFlow 功能文档

## 产品概述

TreeFlow 是一个基于 Node.js + React 的 AI 对话代理系统，核心特色是**分支对话**和**用户自主接入模型**能力。用户可以通过简单的 Token 配置接入任意 OpenAI 兼容的 AI 服务，同时在对话过程中随时创建分支，探索不同的对话路径。

## 数据结构

### 2.1 tokens.json（Token 数据结构）

```json
{
  "tokens": [
    {
      "token": "sk-xxx...",
      "provider": "OpenAI",
      "model": "gpt-3.5-turbo",
      "createdAt": "2024-01-15T08:30:00.000Z",
      "status": "active"
    }
  ]
}
```

**字段说明：**
- `token`: Token 值
- `provider`: 提供商名称（OpenAI/Anthropic/阿里云等）
- `model`: 默认模型名称
- `createdAt`: 创建时间
- `status`: 状态（active/inactive/expired/testing）

### 2.2 topics.json（话题数据结构）

```json
{
  "topics": {
    "topic-id-xxx": {
      "id": "topic-id-xxx",
      "name": "话题名称",
      "createdAt": "2024-01-15T08:30:00.000Z",
      "currentNode": "node-id-yyy",
      "conversationTree": {
        "id": "node-1",
        "message": "用户消息",
        "response": "AI回复",
        "parentId": null,
        "children": []
      }
    }
  }
}
```

**字段说明：**
- `id`: 话题唯一标识
- `name`: 话题名称
- `currentNode`: 当前所在节点ID
- `conversationTree`: 对话树结构（支持多分支）

### 2.3 config.json（配置数据结构）

```json
{
  "currentModel": "gpt-3.5-turbo",
  "ollamaBaseUrl": "http://localhost:11434",
  "ollamaEnabled": false,
  "currentTopic": "default",
  "theme": "dark",
  "providers": {
    "OpenAI": {
      "apiUrl": "https://api.openai.com/v1/chat/completions",
      "method": "POST",
      "headers": {
        "Content-Type": "application/json",
        "Authorization": "Bearer {{token}}"
      },
      "requestBody": {
        "model": "{{model}}",
        "messages": [{"role": "user", "content": "{{question}}"}]
      },
      "responsePath": "choices[0].message.content"
    },
    "default": "OpenAI"
  }
}
```

**字段说明：**
- `currentModel`: 当前使用的模型
- `ollamaBaseUrl`: Ollama服务地址
- `ollamaEnabled`: 是否启用Ollama
- `theme`: 主题（light/dark）
- `providers`: 提供商配置模板

### 2.4 skills.json（技能数据结构）

```json
[
  {
    "id": "translate",
    "name": "翻译",
    "description": "中英互译，自动识别输入语言",
    "icon": "🌐",
    "systemPrompt": "你是一个专业翻译助手...",
    "placeholder": "[翻译] 输入需要翻译的内容"
  }
]
```

## 核心功能详解

### 4.1 话题管理

话题是TreeFlow中组织对话的基本单位。每个话题有独立的对话树，用户可以在不同话题间切换，每个话题的对话历史互不影响。

**创建话题：**
- 生成唯一话题ID（`topic-${timestamp}`）
- 创建空对话树（首次提问时初始化）
- 保存到 topics.json

**切换话题：**
- 更新当前话题配置
- 自动加载该话题的对话历史
- 恢复对话树状态

### 4.2 Token 管理

Token是连接AI服务的凭证。TreeFlow支持：

**自动识别厂商：**
- 根据Token前缀自动识别提供商
- 支持 OpenAI、Anthropic、Google、阿里云等主流厂商

**健康检查：**
- 检测Token是否有效
- 更新Token状态（active/inactive/expired）

### 4.3 分支对话（核心特色）

分支对话允许用户从任意消息节点创建新的对话分支，在不影响主线对话的情况下探索不同方向。

**创建分支：**
1. 用户选择从某个节点分支
2. 系统在该节点下创建新的子分支
3. 后续对话在新分支上进行

**视觉提示：**
- 分支节点有特殊样式标识
- 引用分支显示橙色虚线
- 脑图可视化展示完整对话树结构

### 4.4 脑图可视化

基于 AntV X6 实现的对话树可视化：

**布局：**
- 主流程垂直排列
- 分支水平展开
- 自动计算节点位置

**交互：**
- 划词引用：选中文字创建引用分支
- 节点操作：编辑、复制、删除
- 画布操作：拖拽平移、滚轮缩放
- 小地图：快速定位

### 4.5 Skill 技能系统

预设的AI技能，通过系统提示词实现特定功能：

**使用方式：**
1. 输入 `/` 触发技能选择
2. 选择技能（如"翻译"）
3. 输入内容，AI按技能设定角色回答

**预设技能：**
- 翻译、总结、代码解释、润色、头脑风暴等

## 技术架构

### 5.1 后端架构

```
server/
├── core/                      # 核心业务逻辑
│   ├── agent/
│   │   └── TreeFlowAgent.js   # TreeFlowAgent 主类
│   ├── managers/
│   │   ├── ApiManager.js      # API管理器
│   │   ├── TopicManager.js    # 话题管理器
│   │   ├── TokenManager.js    # Token管理器
│   │   ├── ConfigManager.js   # 配置管理器
│   │   └── SkillManager.js    # 技能管理器
│   └── utils/
│       └── logger.js          # 日志模块
├── server/                    # 服务器层
│   ├── routes/                # 路由定义
│   ├── controllers/           # 控制器
│   └── middleware/            # 中间件
├── data/                      # 数据存储
└── server.js                  # 服务器入口
```

### 5.2 前端架构

```
client/src/
├── App.jsx                    # 主应用
├── components/                # 组件目录
│   ├── layout/                # 布局组件
│   ├── chat/                  # 对话组件
│   ├── mindmap/               # 脑图组件
│   ├── settings/              # 设置组件
│   └── common/                # 通用组件
├── hooks/                     # 自定义Hooks
├── services/                  # API服务
└── utils/                     # 工具函数
```

### 5.3 数据流

```
用户操作 → React组件 → API服务 → 后端API → Agent → Manager → 文件存储
                ↑                                              ↓
                └──────────── 响应数据 ←  JSON ←  业务逻辑  ←────┘
```
